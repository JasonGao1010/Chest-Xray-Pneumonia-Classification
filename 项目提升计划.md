# 肺炎胸部 X 光分类项目提升计划（严格验收版）

> 修订日期：2026-07-10
> 适用目标：《模式识别与机器学习》课程设计最终验收
> 执行原则：先消除会改变结论的错误，再做模型提升；任何结果必须能追溯到固定数据、配置、权重、预测文件和审计报告。

## 1. 审计后的项目判断

现有项目已经具备训练、评价、外部测试、概率校准、错误分析、Grad-CAM 和网页演示的完整工程流程，但原有研究结论不能直接作为最终严格结论。完整性审计发现：Kermany 数据集中，按文件名解析的肺炎患者编号有 170 个同时出现在官方 train 和 test 中。没有发现字节级相同图像跨集合，也没有发现标签冲突；40 份带预测文件的评价 JSON 均可从预测 CSV 精确复算。但同一患者的不同图像跨越训练集与测试集，会破坏样本独立性，并可能使模型利用患者特征。

因此，旧结果的定位调整为：

- Kermany 官方划分结果：保留为“公开目录结构复现基线”，不得称为严格独立内部测试。
- RSNA 结果：患者 ID 唯一、固定患者级划分，无跨集合重复，可作为当前主要外部压力测试。
- NIH 结果：无跨集合重复，但只有 130 张弱标签图像，只能作为补充敏感性分析。
- 原有多轮方案均观察过 Kermany test 和 RSNA test，存在测试集适应风险；下一阶段必须建立新的开发集、内部锁定测试集和最终外部锁定测试规则。
- 现有 51 页报告中第 20--50 页几乎由单图附录构成，不能满足“正文内容达到 50 页”的质量要求，必须重写正文结构。

## 2. 最终研究问题

课程设计不再把“在一个公开划分上取得较高准确率”作为唯一目标，而回答以下四个问题：

1. 患者级数据泄漏会怎样影响胸片肺炎分类结果的可信度？
2. 在无患者重叠的内部划分上，CNN 与 Transformer 的判别、校准和稳定性有何差异？
3. 模型从儿童胸片迁移到 RSNA 外部数据时，性能下降主要表现在哪些指标和错误类型？
4. 统一预处理、合理训练策略和有限域泛化方法，能否提高外部性能且不过度损害内部性能？

建议最终题目：

> 基于深度学习的胸部 X 光肺炎分类：患者级数据审计、可靠性评价与跨数据集泛化研究

## 3. 不可突破的研究边界

- 本项目是图像级正常/肺炎二分类，不是临床诊断系统。
- Kermany 为儿童胸片，RSNA 的人群、标签和采集流程不同，跨域差异不能全部解释为模型缺陷。
- RSNA 的阴性类是非肺炎，不应在医学语义上直接等同于健康正常。
- NIH 标签来自报告文本挖掘，不能与人工精标标签等同。
- Grad-CAM 表示模型贡献区域，不表示真实病灶位置。
- 任何阈值、温度或超参数只能由开发/验证数据确定；锁定测试集只允许最终评价。
- 不以单次最好结果作为提升证据，核心结果必须报告多随机种子和置信区间。

## 4. 数据协议重建

### 4.1 Kermany 患者级重划分

把原 train、val、test 重新汇总，仅依据二分类标签与文件名患者编号分组。相同患者的全部图像必须进入同一集合。建议比例：

| 集合 | 患者比例 | 用途 | 是否允许反复观察 |
|---|---:|---|---|
| development train | 70% | 参数学习 | 允许 |
| development validation | 10% | checkpoint、阈值、温度和超参数选择 | 允许 |
| locked internal test | 20% | 最终内部评价 | 禁止用于调参 |

划分采用标签分层和固定 seed=42，输出患者级 manifest、图像级 manifest、类别/患者统计和 SHA256。正式训练开始后，locked internal test manifest 不再改变。

### 4.2 RSNA 使用规则

- 现有 1707 张镜像可用子集保留患者级 train/val/test。
- RSNA train 与 val 可用于域适配方法开发；RSNA test 冻结。
- 若某方案根据 RSNA test 表现反复修改，它只能算探索结果，最终必须使用新的未参与选择的数据集或预留子集复核。
- 报告中完整披露其并非完整挑战集，以及阴性标签的实际含义。

### 4.3 NIH 使用规则

- 仅做弱标签敏感性分析，不承担模型选择。
- 主文只报告估计值与宽置信区间；不对 26 张 test 的小差异作排序结论。
- 若资源允许，扩大到每类至少 200 张后再进入主要外部表。

### 4.4 数据审计门禁

每次训练前自动检查：

- 患者编号是否跨 split；
- 文件 SHA256 是否跨 split；
- 同一图像是否存在标签冲突；
- manifest 路径是否存在且可读；
- 类别数、患者数和图像数是否与冻结摘要一致；
- 训练目录是否意外包含 test 文件；
- 数据划分文件是否被修改。

门禁失败时不得启动正式实验。

## 5. 统一实验协议

### 5.1 模型范围

主比较只保留三种有代表性的结构：

- DenseNet121：胸片研究中常见的卷积基线，参数量适中；
- ConvNeXt-Tiny：现代卷积结构；
- ViT-B/16：Transformer 对照。

不再增加缺乏明确假设的第四、第五种模型。方法改进优先围绕数据协议、训练稳定性和域泛化展开。

### 5.2 公平比较设置

- 输入尺寸统一为 224×224；
- 评价预处理完全一致；
- 训练增强、优化器、最大 epoch 和 early stopping 规则预先固定；
- checkpoint 只能按 validation loss 或预先指定指标选择；
- 每个核心方案运行 seeds 42、43、44，资源允许时扩到 5 seeds；
- 保存完整配置、环境版本、训练曲线、best/last checkpoint 和预测 CSV；
- 不同模型训练预算与选择机会尽量一致。

### 5.3 可复现性修正

- Python、NumPy、PyTorch 与 CUDA 全部固定随机种子；
- 关闭 cuDNN benchmark，启用 deterministic 模式并记录警告；
- 保存依赖版本、GPU、CUDA/cuDNN 信息；
- 所有结果文件携带 run ID、数据 manifest 哈希、配置哈希和 checkpoint 哈希；
- 汇总表只能从机器结果自动生成，禁止手工复制数字。

## 6. 实验矩阵

### Stage A：无泄漏基线重建

| 编号 | 训练数据 | 模型 | 目的 |
|---|---|---|---|
| A1 | Kermany grouped train | DenseNet121 | 主 CNN 基线 |
| A2 | 同上 | ConvNeXt-Tiny | 现代 CNN 对照 |
| A3 | 同上 | ViT-B/16 | Transformer 对照 |

验收：3 seeds 全部完成；内部 locked test 只在模型与阈值冻结后评价；报告均值、标准差、95% bootstrap CI。

### Stage B：官方划分偏差量化

使用相同模型和尽可能一致的训练设置，对比官方文件级划分与患者级划分。目的不是追求更高分，而是量化数据协议对结论的影响。必须同时报告图像数、患者数、患者重叠和性能变化。

### Stage C：数据捷径与域差异诊断

1. 训练 Kermany/RSNA 来源分类器；若来源容易识别，说明数据域差异明显。
2. 比较原图、去黑边、统一灰度范围和肺野中心裁剪。
3. 做图像尺寸、亮度、对比度、边框比例和文字/标记统计。
4. 对错误样本按数据来源、类别、置信度和图像质量分组。
5. 用遮挡敏感性或 Grad-CAM 检查边缘和非肺区依赖，仅作模型行为分析。

### Stage D：低成本稳健性改进

按单因素消融依次比较：

- 基础增强；
- 胸片适配强度增强（gamma、对比度、轻度噪声）；
- MixUp；
- label smoothing；
- 类别权重与 balanced sampler；
- 统一裁边/灰度预处理。

每次只改变一个主因素。先用 development validation 选择，候选方案不超过两种进入 locked test。

### Stage E：跨域训练

在低成本方案不足时比较：

- Kermany-only；
- RSNA-only；
- 简单混合；
- domain-balanced sampling；
- GroupDRO；
- CORAL 特征对齐。

主要判断采用“双域约束”：外部 balanced accuracy 提高，同时内部 balanced accuracy 的绝对下降不超过 1 个百分点。若未满足，作为负结果保留。

### Stage F：概率与阈值可靠性

- validation 上选择阈值，test 只评一次；
- 报告默认阈值与冻结阈值两组结果；
- temperature scaling 只能拟合 validation；
- 报告 AUROC、AUPRC、accuracy、balanced accuracy、precision、recall、specificity、F1、Brier、ECE、NLL；
- 报告 sensitivity=0.90/0.95 时的 specificity；
- 使用患者级 bootstrap 计算95% CI；
- 多模型比较采用配对 bootstrap，控制多重比较解释。

### Stage G：解释与失败分析

- FP、FN、高置信错误、阈值边界样本分别统计；
- 不只展示“好看的”Grad-CAM，必须包含成功和失败案例；
- 明确热图选择规则，避免事后挑图；
- 对RSNA框标注仅做粗粒度 pointing-game/box overlap 探索，不能称作定位性能；
- 分析正常误报、肺炎漏检和跨域概率漂移。

### Stage H：系统与工程验收

- 网页上传、格式校验、异常处理、CPU推理和警示语测试；
- 权重 SHA256 与 release 附件一致；
- README 命令从干净环境可执行；
- 自动测试覆盖数据审计、指标、校准、患者级划分、推理接口；
- 最终 Git 工作区、远端 refs 和报告版本一致。

## 7. 成功标准与停止规则

“项目提升成功”至少满足：

1. 无患者/图像跨集合泄漏；
2. 核心结果至少3 seeds，方向一致；
3. 外部 balanced accuracy 相对严格基线提高至少3个百分点，或在相近判别性能下明显改善Brier/ECE；
4. 内部 balanced accuracy 下降不超过1个百分点；
5. 95% CI 和配对比较支持差异不是明显的单次随机波动；
6. 第三个未参与调参的数据集方向一致时，才称为可迁移提升。

若连续两轮方法开发都不能越过门槛，停止堆叠复杂方法，把结论收束为数据泄漏审计、跨域失效机制和负结果分析。

## 8. 产物与机器门禁

最终必须形成：

- `results/integrity_audit.json`：数据与结果完整性审计；
- 患者级 split JSON/CSV 与哈希；
- 每个正式 run 的配置、日志、checkpoint 和预测；
- 自动生成的主结果、外部结果、校准与统计比较表；
- 错误样本清单及选择规则；
- 完整课程报告源码与 PDF；
- 报告 claim-evidence 表，逐条对应结果文件；
- 复现说明、环境锁定信息和最终验收记录。

## 9. 执行顺序

1. 完成审计脚本、测试与现有结果复算；
2. 生成患者级 Kermany 新划分并冻结；
3. 重跑三模型严格基线；
4. 完成多 seed、bootstrap 和校准评价；
5. 做来源诊断和低成本预处理消融；
6. 仅在必要时运行 GroupDRO/CORAL；
7. 冻结最终候选并评价内部/外部锁定集；
8. 自动生成表图和证据映射；
9. 重写报告正文并进行两轮严格 review；
10. 编译、数字核对、引用核对和最终复现验收。

本计划替代旧版“通过增加图表和附录达到50页”的思路。最终质量由无泄漏协议、可复算数字、统计不确定性、负结果解释和正文论证密度决定。
