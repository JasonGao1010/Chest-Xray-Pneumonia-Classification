# 肺炎 X 光图像诊断识别项目提升计划

> 2026-07-10 最终验收：提升计划的实验矩阵均已落地。RSNA 镜像可用子集扩容到 1707 张，train 为 560/492、val 为 107/106、test 为 215/227，达到本计划规定的最低外部验证与校准规模。三个模型已完成双数据集评价；DenseNet121 已完成 RSNA-only、简单混合、domain-balanced、冻结分类头、最后 block 解冻和全模型低学习率微调，并同时回测 Kermany。校准、错误分析、Grad-CAM、概率分布和 50 页以上报告均已生成。边界保持不变：RSNA 不是完整挑战数据，NIH 仍是 130 张弱标签压力测试，现有结果不构成临床验证。

## 1. 总体判断

当前项目已经完成一个完整的课程设计版本：数据集准备、模型训练、测试评价、阈值分析、误差分析、Grad-CAM 和网页演示都已经具备。主模型 ViT-B/16 在 Kermany 测试集上的默认阈值结果为 accuracy 0.9006、F1 0.9248、ROC-AUC 0.9710，复现检查已经通过。

后续提升的重点不应只是继续堆模型数量。单数据集上多跑几个模型可以增加表格，但研究问题仍然偏窄。更有价值的升级方向是检验模型在不同来源胸片上的表现：当前模型在 Kermany 儿童胸片上表现较好，但换到其他医院、年龄结构、标注方式和图像格式后，性能是否稳定。这能把项目从“单数据集二分类实验”提升为“模型泛化能力与可靠性分析”。

本计划建议采用五条主线：

- 外部数据集比较：优先扩充 RSNA，做有足够样本量的跨数据集测试和必要的微调对比。
- 外部样本量补强：把 RSNA 从当前小样本压力测试扩展到能支撑结论的规模，避免用几十张样本下强结论。
- 模型体系扩展：增加 2 个有代表性的模型，不做无意义模型堆叠。
- 概率可靠性分析：加入校准指标、可靠性曲线和温度缩放。
- 错误样本分析：围绕高置信错误、正常误报、肺炎漏检、数据来源差异做定量和定性分析。

最终目标是形成一个更完整的研究问题：

> 在儿童胸片肺炎二分类任务上表现较好的深度模型，面对外部胸片数据时是否仍然可靠；模型结构、阈值选择和概率校准会怎样影响误报、漏检和跨数据集泛化？

本计划现在以 50 页以上课程设计报告为最终交付目标。后续所有新增实验都要服务报告论证，不以功能数量为目标。如果某项实验不能进入报告主线，或者只能增加工作量但不能回答老师可能追问的问题，应降低优先级。

报告主线优先级如下：

1. 当前 Kermany 主结果必须稳定复现。
2. 必须把 RSNA 外部样本扩到有说服力的规模，用来回答泛化能力问题。
3. 必须补概率可靠性分析，用来回答模型概率是否可信。
4. 必须补错误样本分析，用来回答误报和漏检来自哪里。
5. 必须做泛化训练策略对照，包括混合训练、domain-balanced sampling 和冻结/解冻微调。
6. 新增模型数量控制在 1 到 2 个，优先 DenseNet121，其次 ConvNeXt-Tiny，不再靠堆模型提高复杂度。

如果时间、算力或数据下载受限，先保 RSNA 样本扩容、外部验证、校准分析和错误分析；模型扩展可以减少，但报告证据链不能断。当前阶段不要再优先新增模型，先解决外部样本量不足。

## 2. 当前基线状态

### 2.1 已完成内容

  - val：16 张，NORMAL 8，PNEUMONIA 8。
  - test：624 张，NORMAL 234，PNEUMONIA 390。
- 主模型：ImageNet 预训练 ViT-B/16，加类别权重。
- 主结果：
  - accuracy：0.9006
  - precision：0.8779
  - recall：0.9769
  - specificity：0.7735
  - F1：0.9248
  - ROC-AUC：0.9710
  - 混淆矩阵：[[181, 53], [9, 381]]
- 已有分析：
  - EfficientNetV2-S 与 ViT-B/16 比较。
  - 默认阈值与测试集阈值扫描。
  - 肺炎子类型统计，包含 bacteria / virus 文件名子类。
  - Grad-CAM 定性解释。
  - 单图上传网页演示。
  - 独立校准集尝试，但没有带来官方测试集提升。

### 2.2 当前短板

- Kermany 验证集只有 16 张，不能支撑稳定阈值选择。
- 当前 RSNA 外部子集只有 88 张，其中 test 只有 40 张，每类 20 张。它只能作为压力测试，不能作为充分外部验证。
- NIH 外部子集只有 130 张，其中 test 只有 26 张，每类 13 张，而且标签来自报告文本挖掘，噪声较高。
- RSNA-only 训练当前只有 40 张 train，Kermany+RSNA 混合训练中 RSNA 增量太小，不能支撑稳定的泛化训练结论。
- 概率校准已有流程，但 Kermany val、RSNA val 和 NIH val 样本都偏少，temperature scaling 只能写成流程验证和限制分析。
- 当前结果已经说明外部泛化下降，但还不能说明哪种训练策略能稳定提高泛化性。

## 3. 推荐总体路线

推荐按“先补足外部样本量，再比较泛化训练策略”的顺序推进。

第一阶段：冻结当前 Kermany 基线，保证报告主结果可复现。  
第二阶段：扩充 RSNA 外部数据，形成有说服力的 train / val / test。  
第三阶段：做 Kermany 训练模型在扩容 RSNA test 上的直接迁移测试。  
第四阶段：做 RSNA-only、Kermany+RSNA 混合训练、domain-balanced sampling 和冻结/解冻微调。  
第五阶段：加入概率校准与可靠性分析，校准只能使用 val，不能使用 test。  
第六阶段：整合错误样本、图像来源差异和 Grad-CAM 分析。  
第七阶段：更新 50 页以上 LaTeX 报告，并同步 README、结果表和图表。

优先级最高的是第二、三、四、五阶段。它们直接对应报告中最容易被追问的点：外部样本是否足够、跨数据来源是否稳定、混合训练是否真的有效、概率是否可信。模型扩展已经够用，不再作为下一阶段主线。

## 4. 数据集扩展计划

### 4.1 首选外部数据集：RSNA Pneumonia Detection Challenge

推荐优先引入 RSNA Pneumonia Detection Challenge。

选择原因：

- 任务与肺炎高度相关，适合当前项目。
- 数据来自 NIH ChestX-ray14 的一部分，并由 RSNA 挑战组织用于肺炎检测。
- 数据包含 DICOM 图像和肺炎区域框，既可以转成二分类，也可以作为后续定位分析材料。
- 比 CheXpert、VinDr-CXR 更容易围绕“肺炎”讲清楚。

使用方式：

- 第一版只做二分类：
  - 有肺炎框或 target=1 的样本记为 PNEUMONIA。
  - target=0 或无肺炎框样本记为 NORMAL / NON_PNEUMONIA。
- 不在第一版做检测框模型。
- 框标注只作为后续解释分析参考，比如看 Grad-CAM 是否大致落在标注区域附近。

建议目录：

```text
data/raw/rsna_pneumonia/
  stage_2_train_images/
  stage_2_train_labels.csv
  stage_2_detailed_class_info.csv

data/processed/rsna_binary/
  train/
    NORMAL/
    PNEUMONIA/
  val/
    NORMAL/
    PNEUMONIA/
  test/
    NORMAL/
    PNEUMONIA/
  metadata.csv
```

第一版可按 patientId 或 imageId 做固定随机划分：

- train：70%
- val：10%
- test：20%
- 随机种子：42
- 划分文件必须保存，不能每次运行重新随机。

样本量要求分两档：

| 用途 | 最低可接受规模 | 更有说服力规模 |
|---|---:|---:|
| RSNA train | 每类 500 张 | 每类 1000 张以上 |
| RSNA val | 每类 100 张 | 每类 200 张以上 |
| RSNA test | 每类 200 张 | 每类 500 张以上 |

最低执行目标是 RSNA train 约 1000 张、val 约 200 张、test 约 400 张。当前 88 张 RSNA 子集只保留为小样本先导实验，不再作为最终外部验证依据。

产物：

- `scripts/prepare_rsna_binary.py`
- `scripts/check_external_dataset.py`
- `data/processed/rsna_binary/metadata.csv`
- `results/rsna_dataset_summary.json`
- `figures/rsna_class_distribution.png`

验收标准：

- 能从原始 RSNA 文件稳定生成二分类目录。
- metadata 中包含原始 patientId、原始标签、二分类标签、split、图像路径。
- 每个 split 的 NORMAL / PNEUMONIA 数量清楚可查。
- 运行检查脚本后，图像总数、类别数、无效图像数写入 JSON。
- RSNA test 至少每类 200 张，否则报告中只能写成“小样本压力测试”。
- RSNA val 至少每类 100 张，否则不能把 temperature scaling 或阈值选择写成稳定结论。

### 4.2 备选外部数据集：NIH ChestX-ray14

NIH ChestX-ray14 可作为第二优先外部数据集。

适用目标：

- 评估模型面对弱标签大规模胸片数据时是否稳定。
- 构建 “Pneumonia vs No Finding” 或 “Pneumonia vs non-Pneumonia” 子集。

风险：

- 标签来自报告文本挖掘，噪声高。
- 多标签样本较多，二分类定义要写得很谨慎。
- 原始数据体量较大，下载和整理成本高。

建议用法：

- 只抽取一个可控子集，不一开始使用全部 112k 图像。
- PNEUMONIA：标签包含 Pneumonia 的样本。
- NORMAL：优先使用 No Finding 样本。
- 控制类别数量平衡，例如每类最多 1500 到 3000 张。
- 明确写成弱标签外部测试，不和人工精标数据等同。
- NIH test 每类至少 200 张后，才适合放入主表做外部弱标签验证；低于这个规模只能作为补充压力测试。

### 4.3 不建议第一轮使用的数据集

CheXpert：

- 优点是规模大、标签体系完整、有不确定标签。
- 问题是下载注册和不确定标签处理更复杂。
- 更适合后续扩展到多标签胸片识别，不适合当前第一轮复杂度升级。

VinDr-CXR：

- 优点是有放射科医生标注和定位框。
- 问题是 DICOM、检测框、多疾病标签都会增加处理复杂度。
- 适合作为后续“定位解释”扩展，不适合作为第一轮主线。

### 5.1 实验问题

核心问题：

> 在 Kermany 数据集上训练得到的模型，直接应用到 RSNA 外部数据时，性能会下降多少？

## 6. 模型扩展计划

### 6.1 新增模型 1：DenseNet121

推荐新增 DenseNet121。

原因：

- DenseNet 在胸片任务中很常见，CheXNet 就基于 DenseNet121。
- 可以作为强 CNN 基线。
- 参数规模相对可控，训练成本低于 ViT-B/16。
- 与 EfficientNetV2-S 形成 CNN 内部对照。

训练设置建议：

- 输入尺寸：224 × 224
- 预训练：ImageNet
- epoch：3 到 5
- batch size：32
- optimizer：AdamW
- learning rate：0.0001 或 0.0003
- weight decay：0.0001
- 类别权重：保留一组加权，一组不加权

产物：

- `configs/densenet121.yaml`
- `models/densenet121_best.pt`
- `results/eval_test_densenet121.json`
- `results/predictions_test_densenet121.csv`
- `figures/confusion_matrix_test_densenet121.png`

### 6.2 新增模型 2：ConvNeXt-Tiny

推荐新增 ConvNeXt-Tiny。

原因：

- ConvNeXt 是现代 CNN 结构，适合和 ViT-B/16 比较。
- 训练和推理成本比大模型更可控。
- 可以形成 “传统 CNN / 现代 CNN / Transformer” 三类对比。

训练设置建议：

- 输入尺寸：224 × 224
- 预训练：ImageNet
- epoch：4 到 6
- batch size：16 或 32，按显存决定
- learning rate：0.00005 到 0.0001
- optimizer：AdamW
- 类别权重：至少跑一组加权

产物：

- `configs/convnext_tiny.yaml`
- `models/convnext_tiny_best.pt`
- `results/eval_test_convnext_tiny.json`
- `results/predictions_test_convnext_tiny.csv`
- `figures/confusion_matrix_test_convnext_tiny.png`

### 6.3 不建议新增太多模型

不建议第一轮增加 ResNet18、ResNet50、MobileNet、Swin、DeiT、ViT-L 等一长串模型。原因很简单：模型数量越多，训练成本和结果解释成本越高，但不一定带来清楚结论。

本项目最合适的模型表述是：

- EfficientNetV2-S：已有 CNN 基线。
- DenseNet121：医学胸片常用 CNN 强基线。
- ConvNeXt-Tiny：现代 CNN。
- ViT-B/16：Transformer 主模型。

这样模型体系已经足够完整。

### 6.4 泛化性训练策略

下一阶段提高泛化性时，优先调整训练策略，而不是继续换更大的模型。

必须比较：

1. 直接迁移：Kermany 训练模型直接测试 RSNA。
2. 简单混合训练：Kermany train 与 RSNA train 合并训练。
3. domain-balanced sampling：每个 batch 中固定比例采样 Kermany 和 RSNA，避免 Kermany 大样本淹没 RSNA。
4. 冻结 backbone 微调分类头：先只训练分类头，检查小外部数据能否稳定适配。
5. 分层解冻：只解冻最后一个 block，再和全模型微调比较。
6. 全模型低学习率微调：作为适配上限，同时检查是否损伤 Kermany test。

建议增强：

- brightness / contrast jitter。
- random affine。
- CLAHE 或直方图均衡对照。
- DICOM window normalization 对照。
- 灰度图转 RGB 的一致处理。

验收标准：

- 每种训练策略都要同时报告 Kermany test 和 RSNA test。
- 如果 RSNA 提升但 Kermany 明显下降，要写成 trade-off，不能只报好看的 RSNA 数字。
- 如果 domain-balanced sampling 优于简单合并，报告中重点解释“域采样比例”而不是夸大模型结构。
- 所有阈值和 temperature scaling 只能在 val 上确定。

## 7. 概率校准与可靠性分析

### 7.1 为什么要做

当前模型输出的是 NORMAL / PNEUMONIA 概率。医学图像任务里，概率是否可信比单纯分类是否正确更重要。一个模型可以有不错的 F1，但概率可能过度自信。外部数据集上，这个问题通常更明显。

### 7.2 新增指标

建议加入：

- Brier score：衡量概率预测和真实标签之间的均方误差。
- ECE：expected calibration error，衡量预测置信度和真实准确率之间的差距。
- MCE：maximum calibration error，可选。
- reliability diagram：可靠性曲线。
- confidence histogram：预测置信度分布。

### 7.3 校准方法

第一轮只做 temperature scaling。扩容后 RSNA val 样本必须足够，否则校准结果只能当流程验证。

流程：

1. 在训练集内部划出 calibration split，或使用外部数据的 validation split。
2. 固定模型参数，只学习一个温度参数 T。
3. 用 T 调整 logits。
4. 比较校准前后的 ECE、Brier score、F1、recall、specificity。

注意边界：

- 不允许用 test set 学习温度参数。
- 如果用 RSNA val 校准，只能报告“在 RSNA val 上校准后对 RSNA test 的结果”。

- RSNA val 每类少于 100 张时，不建议把校准结论写成稳定结论。

产物：

- `src/xray_pneumonia/calibration.py`
- `scripts/analyze_calibration.py`

- `results/calibration_rsna_vit_b16.json`
- `figures/reliability_kermany_vit_b16.png`
- `figures/reliability_rsna_vit_b16.png`

验收标准：

## 8. 错误样本与可解释性扩展

### 8.1 错误样本分类

新增错误样本分析脚本，至少输出：

- false positive：正常样本被判为肺炎。
- false negative：肺炎样本被判为正常。
- high-confidence false positive：错误且预测置信度 >= 0.9。
- high-confidence false negative：错误且预测置信度 >= 0.9。
- borderline samples：肺炎概率在 0.4 到 0.6 之间。

产物：

- `scripts/analyze_errors.py`
- `results/error_cases_vit_b16_kermany.csv`
- `results/error_cases_vit_b16_rsna.csv`
- `figures/error_case_grid_vit_b16_kermany.png`
- `figures/error_case_grid_vit_b16_rsna.png`

### 8.2 外部数据错误对比

新增一个重点分析：

> Kermany 上的高置信错误和 RSNA 上的高置信错误是否有相同模式？

可观察内容：

- 图像亮度和对比度差异。
- 儿童胸片与成人胸片差异。
- DICOM 转换后窗宽窗位或归一化方式差异。
- 正常样本是否被模型误判为肺炎。
- 肺炎样本漏检是否集中在轻症或局部影像改变不明显样本。

这部分要谨慎措辞。只能说模型行为和图像统计差异，不能写成医学诊断结论。

### 8.3 Grad-CAM 扩展

对 DenseNet121 或 EfficientNetV2-S 做 Grad-CAM，对 ViT-B/16 可先不做注意力可视化。第一轮重点放 CNN，因为卷积模型的 Grad-CAM 更自然。

如果使用 RSNA 框标注，可以做轻量对比：

- 对 true positive 样本生成 Grad-CAM。
- 看热区是否大致覆盖 pneumonia box。
- 只做定性展示，不计算严格定位指标。

如果要计算指标，可选：

- CAM 热区 top 20% 区域与 RSNA box 的 overlap ratio。
- 只作为解释分析，不作为主定位结果。

产物：

- `scripts/generate_gradcam_cases.py`
- `figures/gradcam_kermany_error_cases.png`
- `figures/gradcam_rsna_positive_cases.png`

## 10. 分阶段执行计划

### 阶段 0：冻结现有基线

目标：保证当前成果不会被后续实验覆盖。

任务：

验收标准：

- 后续新增结果都写入新文件，不覆盖主结果。

### 阶段 1：RSNA 数据扩容

目标：把 RSNA 数据从小样本 pilot 扩充为能支撑外部验证的二分类格式。

任务：

- 下载 RSNA 数据，优先使用 Kaggle 竞赛原始数据；如认证受限，可使用 Hugging Face 镜像，但必须记录来源。
- 将 DICOM 转为 PNG 或 JPEG。
- 根据标签 CSV 生成二分类标签。
- 固定 train / val / test 划分，按 patientId 保证不泄漏。
- 写入 metadata。
- 检查图像可读性和类别分布。
- 最低样本量：train 每类 500，val 每类 100，test 每类 200。

产物：

- `data/processed/rsna_binary/`
- `data/splits/rsna_binary_seed42.json`
- `results/rsna_dataset_summary.json`

验收标准：

### 阶段 2：当前主模型外部测试

### 阶段 3：新增模型训练

目标：补齐模型族比较。当前 DenseNet121 和 ConvNeXt-Tiny 已经完成，后续只需在扩容 RSNA 上重跑，不建议继续新增模型。

## 13. 风险和边界

### 13.1 数据标签风险

RSNA、NIH、CheXpert、VinDr 的标签来源和定义不完全一致。Kermany 是儿童胸片二分类数据，RSNA 更偏肺炎检测挑战，NIH 是多标签弱标注数据，CheXpert 有不确定标签。跨数据集结果下降不能简单解释为模型变差，也可能来自标签定义和数据分布差异。

处理方式：

- 报告中必须写明每个数据集标签定义。
- 外部结果写成泛化测试。
- 不把外部数据结果直接和 Kermany 同分布测试等同。

### 13.2 测试集信息泄漏风险

不能用 test set 做：

- 阈值选择。
- 温度参数学习。
- 模型选择。
- 数据清洗规则调优。

允许用 test set 做：

- 最终评价。
- 错误样本描述性分析。
- 已固定策略后的结果展示。

### 13.3 医学表述风险

本项目只能作为课程设计和研究训练。模型输出不能写成临床诊断。Grad-CAM 也不能写成病灶定位证据。

推荐表述：

- “模型更倾向于判为肺炎类。”
- “热区显示模型在该区域获得了较高分类贡献。”
- “该结果提示模型可能利用了与胸片结构相关的视觉模式。”

避免表述：

- “模型准确定位病灶。”
- “模型可以代替医生诊断。”
- “高概率说明患者患有肺炎。”

### 13.4 工作量风险

外部数据下载、DICOM 转换、训练新模型都会增加时间成本。若时间不足，优先保留 RSNA 外部测试和校准分析，模型扩展可以减少到 DenseNet121 一个。
