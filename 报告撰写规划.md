# 课程设计报告撰写规划（正文高密度重写版）

> 修订日期：2026-07-10
> 写作目标：形成一份导师能够逐项复核的研究型课程设计报告。正文不少于50页的要求主要由研究问题、数据审计、方法、实验设计、结果和讨论支撑；整页图片、代码清单、复现命令和附录不计入正文页数目标。

## 1. 当前报告必须推倒重写的原因

现有 PDF 虽有51页，但文本提取显示第20--50页多数页面只有一张图和一句说明，正文实际集中在前19页；第9章以后又以追加方式放在参考文献之后，章节顺序和论文逻辑不合格。报告还存在更严重的证据问题：Kermany官方train/test有170个肺炎患者编号重叠，旧报告未披露这一点，并把官方划分结果作为主模型能力依据。

新版报告必须完成以下纠正：

- 参考文献放在全部正文章节之后；
- 章节从研究问题自然展开，不保留“旧报告+新增第9章”的拼接结构；
- 官方Kermany结果降级为公开划分复现，不再充当严格内部验证；
- 新的患者级划分结果成为内部主结果；
- 所有test阈值扫描明确标为事后分析，不能用于选择最终阈值；
- 50页正文不依靠单图占页、空白、重复表述和大段代码；
- 每个关键数字带结果文件来源，每个医学或方法事实带可核验引用。

## 2. 建议题目与中心论点

建议题目：

> 基于深度学习的胸部 X 光肺炎分类：患者级数据审计、可靠性评价与跨数据集泛化研究

中心论点：

> 单一公开目录划分上的高分不足以说明模型可靠。患者级独立划分、外部数据评价、概率校准和错误分析共同决定结论是否可信；本项目的主要工作是建立一套从数据完整性审计到跨数据集评价的可复现实验流程，并如实呈现有效结果、失败结果和适用边界。

## 3. 正文页数与内容密度规则

目标成稿：正文52--60页；前置部分4--5页；参考文献3--5页；附录不设硬性页数。

正文页面应满足：

- 普通文字页建议500--800个中文字符；
- 表格或半页图页面仍需有完整问题、观察和解释；
- 禁止连续两页只有图片；
- 单张图通常不超过页面高度的45%，除非图本身包含多面板且承担核心论证；
- 每个结果小节采用“问题—设置—结果—解释—限制”结构；
- 图片不得重复表达表格已经完整表达的内容；
- 附录图不计入50页正文目标。

建议页数：

| 部分 | 正文页数 | 主要信息 |
|---|---:|---|
| 第1章 绪论 | 4--5 | 背景、问题、研究目标、工作内容 |
| 第2章 理论与相关工作 | 5--6 | 分类模型、迁移学习、域偏移、可靠性、数据泄漏 |
| 第3章 数据集、任务与完整性审计 | 7--8 | 标签、患者、划分、重复、偏差、伦理边界 |
| 第4章 方法与系统设计 | 7--8 | 三模型、损失、训练、校准、解释、系统架构 |
| 第5章 严格实验协议与统计方法 | 5--6 | 门禁、矩阵、指标、CI、多seed、复现 |
| 第6章 内部基线与数据泄漏影响 | 6--7 | 官方划分与患者级划分比较 |
| 第7章 外部泛化与域差异 | 6--7 | RSNA/NIH、来源差异、外部错误结构 |
| 第8章 稳健训练与消融 | 6--7 | 增强、混合、域均衡、GroupDRO/CORAL |
| 第9章 校准、阈值和错误解释 | 5--6 | Brier/ECE、操作点、FP/FN、Grad-CAM |
| 第10章 演示系统与软件验证 | 3--4 | 架构、接口、异常处理、测试 |
| 第11章 综合讨论与威胁 | 4--5 | 发现、负结果、偏差、有效性边界 |
| 第12章 结论 | 2 | 回答RQ，不新增结果 |

合计约60--71页的章节上限，写作时压缩到52--60页正文，不通过附录凑数。

## 4. 章节详细设计

### 第1章 绪论

1.1 研究背景：胸片图像分类在模式识别课程中的代表性。
1.2 问题定义：输入、输出、二分类标签和非临床边界。
1.3 研究动机：单数据集高分、患者泄漏、域偏移、概率不可靠。
1.4 研究问题：列出RQ1--RQ4。
1.5 本文工作：数据审计、严格基线、外部验证、稳健训练、可靠性分析、演示系统。
1.6 章节安排。

本章不得提前堆所有结果。贡献表述用“完成/比较/审计/分析”，不用“显著提升、临床可用”。

### 第2章 理论基础与相关工作

2.1 胸片二分类的基本流程。
2.2 卷积网络：DenseNet与ConvNeXt的特征复用和现代卷积设计。
2.3 Vision Transformer：patch、self-attention与迁移学习。
2.4 类别不均衡：加权交叉熵、采样及其副作用。
2.5 数据泄漏与患者级划分：为什么同一患者不能跨集合。
2.6 数据域偏移：人群、设备、体位、标签定义和图像处理差异。
2.7 判别与校准：AUROC/AUPRC、Brier、ECE、temperature scaling。
2.8 模型解释：Grad-CAM能说明什么、不能说明什么。
2.9 相关研究不足与本文切入点。

要求：引用原始模型论文、数据集论文和报告规范；不以二手博客支撑关键方法。

### 第3章 数据集、任务定义与完整性审计

3.1 Kermany数据来源、纳入标准与标签。
3.2 RSNA数据来源、检测标签转二分类规则与镜像子集限制。
3.3 NIH弱标签子集与多标签处理。
3.4 图像、患者和事件数统计，区别“图像数”与“患者数”。
3.5 原官方划分审计方法：文件名患者ID、SHA256、标签冲突。
3.6 审计结果：170个肺炎患者ID跨train/test；无精确图像重复。
3.7 患者级重划分算法与冻结manifest。
3.8 数据预处理与质量检查。
3.9 数据集间差异与潜在混杂。
3.10 数据伦理和适用边界。

核心表：三个数据集的图像/患者/类别/标签来源；官方划分重叠矩阵；新划分统计；数据差异表。核心图最多3张，均为多面板。

### 第4章 方法与系统设计

4.1 总体流程图。
4.2 输入表示、resize、RGB转换和归一化。
4.3 数据增强及医学合理性。
4.4 DenseNet121。
4.5 ConvNeXt-Tiny。
4.6 ViT-B/16。
4.7 损失函数与类别处理。
4.8 迁移学习、冻结和分层解冻。
4.9 混合训练与域均衡。
4.10 GroupDRO/CORAL（若执行）。
4.11 温度缩放和阈值冻结。
4.12 错误样本与Grad-CAM。
4.13 网页演示系统数据流。

公式只保留能在实验中实际使用的内容：softmax、加权交叉熵、指标、Brier、ECE、temperature scaling和域目标。

### 第5章 实验协议与统计方法

5.1 硬件、软件与依赖。
5.2 数据门禁和manifest哈希。
5.3 开发集、validation和locked test职责。
5.4 实验矩阵与预先假设。
5.5 公平比较和计算预算。
5.6 多随机种子。
5.7 点估计、标准差与患者级bootstrap CI。
5.8 配对比较与多重比较解释。
5.9 阈值与校准协议。
5.10 结果追溯和复现流程。

本章要让导师能判断：有没有用test调参、不同模型是否公平、差异是否超过随机波动。

### 第6章 内部基线与数据泄漏影响

6.1 官方划分复现结果。
6.2 患者级无泄漏三模型结果。
6.3 多seed稳定性。
6.4 官方与严格划分的性能差异。
6.5 混淆矩阵与类别错误。
6.6 预训练与随机初始化对照。
6.7 本章结论：哪些旧结论保留，哪些被修正。

不能只报最高分。主表必须包含均值、标准差、CI、患者数和图像数。

### 第7章 外部泛化与数据域差异

7.1 外部评价问题。
7.2 Kermany到RSNA直接迁移。
7.3 Kermany到NIH弱标签敏感性分析。
7.4 来源分类与数据捷径诊断。
7.5 亮度、裁剪、边框和概率分布差异。
7.6 外部FP/FN结构。
7.7 外部结果的标签与人群解释边界。

RSNA主表不得把阴性样本简单写成临床正常；NIH小样本差异不作排名。

### 第8章 稳健训练与消融实验

8.1 基础训练方案。
8.2 预处理消融。
8.3 数据增强消融。
8.4 类别权重与采样。
8.5 MixUp/label smoothing。
8.6 RSNA-only、简单混合和域均衡。
8.7 GroupDRO/CORAL。
8.8 双域收益与代价。
8.9 失败方案和停止规则。

每个消融都写明唯一变化、验证集选择依据和test是否锁定。负结果占有正式位置，不删除。

### 第9章 概率可靠性、阈值和模型解释

9.1 判别能力与概率可靠性的区别。
9.2 内部校准。
9.3 外部校准。
9.4 validation冻结阈值与默认阈值。
9.5 固定敏感度操作点。
9.6 FP、FN和高置信错误。
9.7 Grad-CAM选择规则与结果。
9.8 对可靠性结论的限制。

可靠性图使用多面板，一页同时容纳曲线、样本分布和数值摘要，避免一图一页。

### 第10章 演示系统与软件质量

10.1 需求和边界。
10.2 前后端与推理流程。
10.3 模型加载和预处理一致性。
10.4 输入校验、错误处理和安全提示。
10.5 自动测试与复现。
10.6 演示结果及其非临床定位。

网页截图最多半页，不用多张截图扩页。

### 第11章 综合讨论与有效性威胁

11.1 对四个研究问题逐项回答。
11.2 患者级泄漏对既有结果的影响。
11.3 模型结构差异与数据域差异谁更重要。
11.4 稳健训练为何成功或失败。
11.5 内部、外部、构念和统计结论有效性。
11.6 数据选择、弱标签和年龄差异。
11.7 课程设计贡献与临床研究之间的距离。
11.8 后续工作。

### 第12章 结论

按RQ1--RQ4各用一段回答，只引用已经出现的结果，不引入新实验或新引用。最后一段说明工程交付和适用边界。

## 5. 图表控制

正文建议控制在18--24张图、20--28张表。图表必须承担问题回答：

- 数据统计尽量用表；
- 多模型结果用表+一张CI森林图；
- 混淆矩阵合并为多面板；
- 可靠性曲线合并为多面板；
- 错误样本每类固定数量并说明抽样规则；
- 训练曲线只展示有解释价值的代表性run，其余放附录；
- 单独的29张证据图不得继续各占一页。

## 6. 引用与证据规范

- 数据集来源引用原始数据页或原始论文；
- 模型引用原始论文；
- 医学AI报告规范参考TRIPOD+AI，风险讨论参考PROBAST+AI；
- 每条引用实际支撑相邻陈述，不用堆引用；
- DOI、作者、题名、年份逐条核验；
- 结果数字不依赖文献，而依赖本项目机器产物；
- 建立claim-evidence表：章节、论断、指标、结果文件、图表、限制。

## 8. 完稿门禁

提交前逐项通过：

1. 正文页数不含封面、目录、参考文献、附录仍达到课程要求；
2. 不存在连续大图凑页；
3. 目录、图表编号、交叉引用无错误；
4. 所有主结果可从CSV/JSON复算；
5. 报告数据量与manifest一致；
6. 训练/验证/test职责表述一致；
7. 引用100%可核验；
8. 摘要、正文、结论数字一致；
9. 所有负面边界均保留；
10. PDF无溢出、空白页、低清图和字体问题；
11. 自动测试、完整性审计和复现命令通过；
12. 由“挑剔导师视角”至少进行两轮review并逐项关闭问题。

## 9. 文件结构

新版报告采用分文件LaTeX：

```text
reports/thesis/
  main.tex
  sections/
    00_abstract.tex
    01_introduction.tex
    02_background.tex
    03_data_audit.tex
    04_methods.tex
    05_protocol.tex
    06_internal_results.tex
    07_external_results.tex
    08_robustness.tex
    09_reliability.tex
    10_system.tex
    11_discussion.tex
    12_conclusion.tex
  tables/
  refs.bib
```

Markdown可保留为可读版本，但LaTeX是最终排版真源。旧的追加式 `course_report.tex` 不再继续堆补丁。
