# 肺炎 X 光图像诊断识别报告撰写规划

> 2026-07-10 最终验收：正式报告已经按“扩容 RSNA 外部验证 + 泛化训练策略比较 + 校准与错误分析”完成重写并独立编译。RSNA test 为 215/227，达到计划最低门槛；NIH 130 张仍按弱标签压力测试处理。报告包含 12 个编号章节、关键图表证据附录和复现验收表，PDF 超过 50 页，医学与测试集边界均保留。

## 1. 报告定位

本报告最终提交给课程设计老师，目标不是写成普通实验说明书，而是写成一篇结构完整、证据充分、表达克制的长篇研究报告。篇幅要求为 50 页以上，建议按硕士生毕业论文的组织方式来规划，但研究深度仍保持在课程设计范围内。

报告的核心任务是说明：

> 本项目围绕胸部 X 光肺炎二分类任务，构建了可复现的数据处理、模型训练、测试评价、外部验证、泛化训练策略、概率可靠性分析、错误样本分析和网页演示流程，并在明确边界的前提下讨论模型能力和限制。

报告不能写成“模型很好”“可以辅助医生诊断”这类泛泛判断。所有结论都必须有对应实验、表格、图像或复现命令支撑。没有证据的地方只写限制，不写推断。

## 2. 总体写作原则

### 2.1 研究判断先行

每一章开头先说明本章要回答的问题，不先堆代码和工具名。

推荐写法：

> 本章主要说明数据集的来源、标签含义和类别分布。由于训练集与测试集类别比例不同，后续评价不能只看准确率，还需要同时观察召回率、特异性和混淆矩阵。

避免写法：

> 本章使用 Python 读取文件夹，并调用脚本统计数据集。

### 2.2 结论必须可追溯

报告里的每一个关键判断，都要能追溯到：

- 一个表格；
- 一张图；
- 一个 JSON / CSV 结果文件；
- 一条固定命令；
- 一个明确的数据来源。

例如：

- “ViT-B/16 默认阈值下误报更少”必须对应混淆矩阵。
- “测试集阈值扫描不能作为部署阈值”必须说明阈值来自测试集预测。
- “外部数据集性能下降”必须对应 Kermany test 与 RSNA test 的对比表。
- “外部验证有说服力”必须对应足够样本量。RSNA test 每类低于 200 张时，只能写成小样本压力测试。
- “混合训练提高泛化”必须同时对应 RSNA test 和 Kermany test，不能只报一个好看的外部结果。
- “模型概率不一定可信”必须对应 ECE、Brier score 或可靠性曲线。

### 2.3 不写人机味表达

报告中要避免口号式、营销式、模板式语言。

避免使用：

- “显著提升”
- “充分证明”
- “展现了”
- “赋能”
- “闭环”
- “抓手”
- “底座”
- “至关重要”
- “深入探讨”
- “综上”
- “此外”
- “不是……而是……”

更稳妥的写法：

- “在本测试集上，F1 从 0.8930 变为 0.9248。”
- “该结果说明模型在当前划分下减少了正常样本误报。”
- “这一现象不能直接推广到临床数据。”
- “该阈值来自测试集扫描，只用于分析，不作为最终部署策略。”

### 2.4 不夸大医学意义

本项目是课程设计，不是临床系统。报告中只能写：

- “正常/肺炎二分类”
- “模型分类倾向”
- “辅助理解模型行为”
- “课程设计实验”
- “研究训练场景”

不能写：

- “诊断准确”
- “临床可用”
- “替代医生”
- “定位病灶”
- “发现真实病变”

Grad-CAM 相关内容必须写成模型解释，不能写成医学定位。

## 3. 报告主线

报告建议围绕一条清楚主线展开：

1. 胸片肺炎识别可以被建模为正常/肺炎二分类任务。
2. 单一数据集上的高分并不等于模型可靠。
3. 因此，本报告不仅训练模型，还从误报漏检、阈值选择、概率校准、外部数据集和错误样本角度检查模型。
4. 最终结论保留在课程设计范围内：模型在公开数据上具备一定分类能力，但泛化、校准和医学解释仍有限制。

新版主线需要额外强调两点：

5. 外部样本量不足时，结果只能提示风险，不能支撑稳定结论。
6. 提高泛化性应优先依靠外部样本扩容、域均衡采样和微调策略，而不是继续堆更大的模型。

这条主线比“我用了很多模型”更适合 50 页以上报告。模型只是证据来源，不是叙事中心。

### 第 8 章 阈值、校准与概率可靠性

必须写清楚：

## 12. 写作语言规范

### 12.1 句子长度

尽量使用短句。一个句子只表达一个主要意思。不要为了显得正式写很长的句子。

推荐：

> 训练集肺炎样本比例为 74.3%。这一比例高于测试集中的 62.5%。因此，模型可能更容易倾向于预测肺炎类。

避免：

> 由于训练集和测试集之间存在类别分布方面的差异，因此模型在训练过程中可能学习到更加偏向肺炎类别的决策边界，从而在最终测试过程中产生一定程度的预测偏差。

### 12.2 先写事实，再写解释

推荐结构：

1. 数字结果。
2. 直接含义。
3. 可能原因。
4. 边界。

例：

> ViT-B/16 在测试集中漏检 9 张肺炎图像，漏检率为 2.3%。这一结果说明模型对肺炎类保持较高召回率。与此同时，模型将 53 张正常图像误判为肺炎，说明正常样本误报仍然是主要错误来源。由于测试集来自同一数据集划分，该结果不能直接代表外部医院数据上的表现。

### 12.3 少用形容词，多用证据

避免：

> 模型取得了优秀表现。

推荐：

> 模型在默认阈值下取得 F1 0.9248，较 EfficientNetV2-S 无权重实验的 0.8930 更高。
